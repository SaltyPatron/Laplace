FIX 6 — Elo used as a Glicko-2 witness-count multiplier (EloGames)
================================================================================
This one came from the user directly correcting the approach mid-session ("ELO != glicko2"),
not from the original audit.
Change: removed `EloGames(elo)` entirely from `ChessPgnDecomposer.cs` (previously scaled the
MOVE edge's `games`/`moveChoiceGames` witness count by a linear rescaling of the player's raw
Elo number, `(elo-600)/200` clamped 1-12 — a category error, since Elo and Glicko-2 are
different rating systems and the player's real Elo is already losslessly captured by the
existing, separate `HAS_RATING` attestation, `Rating()` at ChessPgnDecomposer.cs:182-186,

outcome-type bonus, unrelated to Elo). `moveChoiceGames` argument is no longer passed at all
(AppendMoveEdge defaults it to `games`). Also removed the equivalent ThinkFactor-based scaling
of `moveChoiceGames` for the same reason (think-time is separately, losslessly captured via
HAS_CLOCK/HAS_THINK_CLASS already) — same principle extended per the user's explicit
"same with the clock" follow-up. Grepped Chess.Service afterward for any other
Elo/CpLoss/think-time-into-witness-count pattern — none found (ChessReviewIngest's
QualityGames/ReviewWitnessWeight and the NAG-based MOVE_QUALITY path were already flat
constants, not derived per-instance).

FIX 7 — UciEngine.cs: unimplemented stop, unbounded go-depth, malformed-FEN crash
================================================================================
Change: `go` now runs the search on a background `Task` (via a new `StartSearch`) instead of
blocking `Handle()`, so `stop` (implemented as `_searchCts.Cancel()` + a bounded `.Wait(2000)`)
can actually interrupt an in-flight search — `Search.Think` gained an optional
`CancellationToken` parameter, checked inside the existing `TimeUp()` gate. `go depth N` (no
other time control) now also gets a 120,000ms hard ceiling (was `int.MaxValue`) as a
belt-and-suspenders safety net — this is the exact invocation `CutechessRunner.cs:58` drives
(`tc=inf/depth={depth}`). `SetPosition` now builds a local board and only commits it to
`_board` after successfully parsing, inside a try/catch for `FormatException`, instead of
mutating `_board` directly and letting a bad FEN propagate an unhandled exception that killed
the whole process. Added a public `WaitForIdle(timeoutMs)` so a synchronous caller (a test
harness, or any non-interactive embedder) can deterministically wait for an in-flight
background search.
REAL REGRESSION CAUGHT AND FIXED DURING THIS SESSION'S OWN VERIFICATION: first
`dotnet test app/Laplace.Chess.Uci.Tests` run after this change: "Failed: 5, Passed: 3, Total:
8" — every test that sends "go ..." and then immediately inspects the output for "bestmove"
failed with "Sequence contains no matching element", because `go` no longer blocks. Root cause:
the existing `Run()` test helper assumed synchronous `Handle()` behavior that this fix removed.
Fixed by calling the new `WaitForIdle()` in the test helper after sending all commands.
Re-ran: "Passed! - Failed: 0, Passed: 8, Skipped: 0, Total: 8".
Then added two NEW tests directly exercising the actual new behavior (not just re-confirming
old behavior still works):
  - `MalformedFen_DoesNotThrow_AndKeepsPriorPosition` — sends a real invalid FEN mid-game,
    asserts the engine doesn't throw/die (isready still answers) and the position is still
    "after 1.e4", not corrupted.
  - `Stop_DuringUnboundedDepthSearch_ReturnsWellBeforeTheSafetyCeiling` — sends "go depth 64"
    (no time control, relies entirely on stop/the 120s ceiling), sleeps 50ms, sends "stop",
    asserts a bestmove appears and the whole round-trip took under 5 seconds — i.e. asserts
    "stop" is what ended it, not the 120-second ceiling.
FINAL VERIFIED: `dotnet test app/Laplace.Chess.Uci.Tests -c Release` ->

FIX 9 — chess HTTP endpoints: path traversal + unbounded concurrent jobs
================================================================================

`ChessGameFetcher.Sanitize(user)`/`Sanitize(site)` (made `internal`, was `private`) instead of
