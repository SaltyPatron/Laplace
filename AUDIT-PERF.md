### [CRITICAL/CONFIRMED] layer-altitude — nearest_neighbors_4d recomputes angular distance via a per-candidate SPI round-trip instead of inside the kNN query.
- **Where:** `extension/laplace_substrate/src/graph_geometry_reads.c:126`  ·  hotness: Per nearest-neighbors query (interactive inference read); 200-2000 SPI sub-executions per call dominate latency. The numeric work is trivial; the SPI re-entry is the cost.

This is exactly the pattern already used in 26_generation.sql.in:668 (laplace_angular_distance_4d applied set-based inside a CTE).

### [CRITICAL/CONFIRMED] simd — project_token inner GEMV loop (E_row . W) uses scalar neumaier_add; the running-compensation dependency serializes the loop and blocks the vectorizer. This is the single hottest kernel: it runs over vocab x head_dim x d_model for both Q and K projection.
- **Where:** `engine/synthesis/src/qk_pairs_threshold.cpp:27`  ·  hotness: Dominates. project_token is called vocab times for keys + (q1-q0) times for queries, each doing head_dim dot products of length d_model. - **Better:** This is plain GEMV: proj = W (head_dim x d_model) * E_row (d_model). Replace the hand-rolled per-row dot loop with cblas_sgemv / cblas_dgemv (MKL is already linked under LAPLACE_HAS_MKL) or batch all tokens as cblas_dgemm (E[vocab x d_model] x W^T). Numerically, a blocked (per-lane) Neumaier with a final cross-lane compensation preserves accuracy while exposing 4-8 way ILP.
