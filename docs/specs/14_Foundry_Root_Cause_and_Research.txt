substrate's riches don't reach the synthesized model — with literature + prescriptions
================================================================================
Date: 2026-07-05. Method: direct read of the Mold-A-Model synthesis
(FoundryCommands.SynthesizeMoldAModelAsync + FoundryExport basis/factor path),
the 2026-07-05 full-reseed live timings, and a targeted literature pass (links in

The one-sentence answer to "what is missing": the synthesis currently uses ~3 of the
substrate's ~12 information channels (eff_mu edge weights, relation planes,
PRECEDES adjacency), projects them all through ONE shared hub-dominated basis,
stacks layers that are LINEAR in content — and every named failure (no
conversation, mashed heads, mashed layers) is a theorem-grade consequence of
those three facts, each with a named literature counterpart and a named fix.

SECTION 1 — WHAT THE SYNTHESIS DOES TODAY (grounded, FoundryCommands.cs ~979-1400,
FoundryExport.cs ~923-1530)
================================================================================
  1. Vocab: tokenizer-derived word tokens + byte floor (word crawl; the 937-vocab
     starvation class from 09 is a property of this choice, not of the substrate).
  2. Basis E: UNION of all operator planes → ONE affinity matrix → raw SVD
     (BuildBasisAffinity: A[x,y] += w symmetrized, TensorSvdTruncate, rank cap =
     env LAPLACE_FOUNDRY_BASIS_RANK default 256), rows L2-normalized, last dim =
     bias. S³ anchors are passed and used for Procrustes orientation
     (BasisStats.ProcrustesResidual), NOT as the graph normalization.
  3. Per layer, per operator key (relation:X / metric:frechet / trajectory /
     unary): m = ProjectOperator(R, plane) (project into the CURRENT residual
     representation R); mResid = m − I; Q/K = Factor(m, rank ≤ headDim);
     V/O = Factor(mResid, transposed); FFN up/down = Factor(mResid, rank ≤ ffn).
  4. Residual construction: update += scale·(M−I)·R summed over ALL of the
     layer's operators, then R ← RMSNorm(R + update). scale = nLayers^-0.25.
  5. Norms: every input/post-attention layernorm weight tensor = 1.0
     (Array.Fill(vals, 1.0f)) — no per-dimension gain anywhere.
  6. FFN gate: constant-column trick (gateZ=6.0 bias column; upGain =
     1/silu(gateZ) exactly cancels it) — the gate NEVER depends on the token.
  7. lm_head: ONE plane (trajectory/PRECEDES stride): lmHead[y] = Σ_x w·R_final[x],
     in-degree idf, row-normalized. Off the composed state (good), but the plane
     is word-adjacency-within-documents (bigram-flavored, tier-2 flat).
  8. "context" head: identity-scaled QK (raw embed similarity), identity V —
     copies similar-token content forward; the only sequence machinery present.
  9. Glicko: plane values carry the folded edge weight (eff_mu-derived). rd,
     volatility, witness_count, source, context, outcome polarity: never read.
 10. Salience ranks: used to GROUP relations into head families (rankNames:
     "taxo+part", "causal+seq", "equivalence") — never as weights or priors.

SECTION 2 — THE MASHING: five mechanisms, each code-grounded + literature-named
================================================================================
M1. THE STACK IS LINEAR IN CONTENT (the single biggest cause).
    Every synthesized weight is a linear factor; the one nonlinearity a transformer
    layer owns (the FFN gate) is constant by construction (§1.6). A stack of
    linear layers collapses to ONE linear map — depth adds smoothing, never
    composition. Dong/Cordonnier/Loukas (ICML 2021) prove the sharpened version:
    pure attention (no MLP nonlinearity, no skip) converges to rank-1 "token
    uniformity" DOUBLY EXPONENTIALLY with depth; the components that provably
    stop it are exactly skip connections (we have) and REAL MLPs (we don't).
    The observed FAITHFUL collapses (them↔they 2-cycle, hub attractors) are this
    theorem happening on schedule. → Prescription P2.

M2. ALL HEADS SHARE ONE SUBSPACE; RELATIONS RE-MASH IN THE RESIDUAL STREAM.
    Every operator is projected through the SAME R and truncated to its own top
    singular directions — but those directions substantially COINCIDE (they are
    the union graph's dominant/hub directions), so heads become near-parallel
    perturbations of the same subspace. And V/O factors all write back into the
    same unallocated residual dimensions — no per-head orthogonal allocation, no
    Gram-Schmidt across head bases (named in 09's toolkit, never implemented).
    The knowledge-graph literature learned this lesson a decade ago: a
    multi-relational graph is a 3-TENSOR (subject, relation, object); collapsing
    relation slices into one matrix loses the relational structure (RESCAL,
    Nickel 2011), and every serious multi-relational GNN keeps RELATION-SPECIFIC
    transforms (R-GCN, Schlichtkrull 2018; r-GAT). The substrate's 181 relation
    [count superseded 2026-07-19 -> 203; docs/INVENTORY.md is the count authority]
    types are the tensor slices; the synthesis re-flattens them twice (union basis +
    summed residual). → Prescriptions P3, P4.

M3. THE LAYER UPDATE IS POWER ITERATION TOWARD THE HUB EIGENVECTOR.
    R += Σ_ops scale·(M−I)·R, then magnitude-only RMSNorm, repeated per layer =
    diffusion toward the summed operator's dominant eigenvector = GNN
    OVERSMOOTHING, verbatim: repeated graph convolution makes node embeddings
    indistinguishable (Li/Han/Wu 2018; Oono & Suzuki 2020). The GNN remedies map
    one-to-one onto what the synthesis lacks: PairNorm (preserve total pairwise
    distances after each propagation — a normalization, cheap), mean-subtraction
    (recenters the iteration onto the FIEDLER vector, i.e. structure instead of
    degree), residuals (provably mitigate — present), DropEdge (stochastic — the
    deterministic analogue is per-layer operator SCHEDULING rather than all-ops-
    every-layer). RMSNorm controls norm, not DIVERSITY; nothing in the loop
    preserves pairwise separation. → Prescription P4.

M4. [CORRECTION 2026-07-08 — STALE, verified against code (doc 18 §6 C-1): the live
    default basis IS the normalized-Laplacian eigenmap (eigenmaps.cpp:56, Spectra low
    eigenvectors, trivial vector dropped); BuildBasisAffinity raw-SVD is dead code
    (affRaw hardcoded null), and hidden='auto' spectral-rank sizing IS implemented
    (FoundryCommands.cs:1080-1091). P1 is effectively done. Text below preserved as
    originally written.]
    THE BASIS ITSELF IS HUB-DOMINATED FROM STEP ONE.
    BuildBasisAffinity does raw SVD of the summed affinity A. The top singular
    directions of an UN-normalized affinity track degree — hubs ("person",
    "them") own the leading coordinates of every token's embedding before any
    layer runs. 09 specifies the NORMALIZED Laplacian eigenmap
    (L = I − D^-1/2 W D^-1/2, LOW eigenvectors = the colony at rest); the code
    ships the un-normalized, top-directions version. Graph-transformer practice
    (LapPE — Dwivedi & Bresson 2020; GraphGPS) uses exactly the low Laplacian
    eigenvectors as structural coordinates, and its known weakness — sign/basis
    ambiguity of eigenvectors (the Laplacian-canonization problem) — is a
    weakness the substrate DOESN'T have: Procrustes-to-S³-anchors provides the
    canonical orientation the literature has to hack around. Also: basis rank is
    an env var (256), not the eigengap — 'hidden=auto' (12's spectral-rank slot)
    is unimplemented. → Prescription P1.

M5. THE GLICKO TUPLE IS TRUNCATED TO A SCALAR; TENSION≠CERTAINTY IN THE SYNTHESIS.
    Only the folded weight reaches a plane. rd (per-edge uncertainty →
    temperature), witness_count (evidence mass → sharpening/saturation),
    volatility (stability → decay), source trust (scoped synthesis runs), outcome
    polarity (refuting vs supporting) are all dropped. Consequence: a
    single-witness rumor edge and a 10k-witness taut strand differ only in one
    number, so hubs with MANY WEAK edges outweigh spokes with FEW TAUT ones in
    every head's attention pattern — the "slack strand" pulls as hard as the
    tight one. 12 §Family-A names the full mapping (rd→temperature,
    witness→mass, volatility→update signal); none of it is wired.
    → Prescriptions P5, P8.

SECTION 3 — WHY NO CONVERSATION, SPECIFICALLY
================================================================================
Conversation is a TIER-3 phenomenon synthesized from a TIER-2-FLAT model. Inventory:

C1. VOCAB: word-crawl + byte floor starves; the closed vocabulary the substrate
    actually owns is tier-0 (assigned codepoints under the pinned UCD —
    deterministic, zero OOV, ~155K assigned; full codespace 1,113,869 live).
    Tier-floor law means single-codepoint words ARE their codepoints — which is
    why FAITHFUL "worked for Japanese": kanji codepoint = word entity = same id,
    so codepoint bigrams were ALREADY word bigrams carrying OMW-ja/Tatoeba
    consensus. Live proof (2026-07-05 DB): tier-3 sentence 事件のことなんだけど...
    decomposes directly to tier-1 grapheme constituents — for CJK the ladder
    already routes sentences onto grapheme/codepoint units. English needs the
    composition layers (C5) to get the same effect. Field parallel: Meta's Byte
    Latent Transformer (2024) reaches token-model quality from raw bytes by
    LEARNING dynamic patches + a hierarchical encoder — the trained
    approximation of tiers. Consensus-derived merges (witness-weighted word
    entities as extra tokens) are the substrate-native BPE.
C2. NO REAL SEQUENCE MACHINERY: the only positional signal is the identity
    "context" head + a trajectory STRIDE plane. The substrate owns two canonical
    positional codes, both unused: hilbert_index (global content-locality 1-D
    address → absolute PE) and trajectory ordinal (exact within-sequence
    position → sequence PE). Without them, prefix mixing cannot represent order
    — no discourse state, no anaphora, no "what did you just say".
C3. THE LM_HEAD SPEAKS DOCUMENT-ADJACENCY, NOT DIALOGUE. One word-tier PRECEDES
    plane. [CORRECTION 2026-07-08 (doc 18 §6 C-2): the claim that a turn/response
    plane "is sitting in consensus, unpoured" is WRONG. OpenSubtitles emits ONLY
    cross-language IS_TRANSLATION_OF line pairs (OpenSubtitlesDecomposer.cs:55-57);
    Tatoeba only translations; no sentence→sentence PRECEDES exists anywhere in
    consensus. Sentence ORDER survives in tier-4 document trajectories, so the
    dialogue plane is a CALCULATED plane over witnessed trajectories
    (sentence_order — remediation plan Phase 4), not an unpoured consensus asset.]
    A conversational lm_head = completion operator over a TIER-3 sentence-order
    bridge composed with the tier-2 continuation plane.
C4. NO CONDITIONING: the serving path converses because recall_*_response does
    intent routing + templated realization server-side. The GGUF gets no
    analogue of intent, persona, or scope. The mold-native mechanism exists on
    paper (12: filter attestations by source/context → re-fold → scoped model)
    and is the "custom-tailored" product feature — unwired.
C5. NO COMPOSITION LAYERS: with a codepoint/merge vocab, the first layers must
    IMPLEMENT the tier ladder (grapheme/word membership — already witnessed
    in trajectories and containment edges, i.e. a KNOWN operator to mold, per
    BLT's local-encoder role). Blocked by M1 (linear layers can't gate
    composition) — this is where M-fixes and C-fixes meet.

SECTION 5 — PRESCRIPTIONS (ordered; each kills a named mechanism; every one is
a deterministic construction, no training anywhere)
================================================================================
P1 BASIS: normalized-Laplacian eigenmap (D^-1/2 W D^-1/2, Lanczos, LOW
   eigenvectors), hidden dim = eigengap ('auto'), Procrustes-to-S³ canonical
   orientation. Kills M4 at the root — hubs stop owning the coordinate system.
P2 NONLINEARITY: make the FFN gate a real function of content. Mold-native
   candidates, no gradients: gate rows from band-membership / highway bits
   (which relation families this token participates in — a genuine step
   function), or containment-mask gating (token fires its composition FFN only
   inside its containers). Minimum viable first step: stop canceling silu.
   Kills M1; unblocks C5.
P3 HEAD ALLOCATION: block Gram-Schmidt across head value/output subspaces so
   each relation band OWNS an orthogonal residual slice; scale o_proj by
   salience rank (the 13 bands become head-importance priors, as designed).
   Kills M2's write-collision half. R-GCN/RESCAL is the precedent: relation-
   specific subspaces are what keep a 3-tensor from flattening.
P4 LAYER DISCIPLINE: (a) PairNorm-style pairwise-distance preservation or
   mean-subtraction after each diffusion step (recenters onto Fiedler/structure
   — the oversmoothing literature's cheapest fix); (b) SCHEDULE operators by
   tier instead of all-ops-every-layer: composition ops in early layers,
   taxonomic/semantic mid, discourse/sequence late — the tier ladder AS the
   layer schedule (12's "LAYER = a hop/tier"). Kills M3; realizes tiers.
P5 GLICKO-COMPLETE WEIGHTS: w_edge = eff_mu · exp(−κ·rd) · sat(witness_count)
   (sat = log1p or the fold's own mass law); volatility as decay on stale
   planes; refuted/low-outcome edges as NEGATIVE attention (signed operator).
   Kills M5 — "tension = certainty" finally reaches the weights.
P6 CONVERSATIONAL OPERATORS + VOCAB: tier-0 closed floor (assigned codepoints,
   pinned UCD) + consensus-derived merges; synthesis the TIER-3 turn/response plane
   (OpenSubtitles/Tatoeba) as the lm_head's discourse component alongside the
   tier-2 continuation plane. Kills C1/C3.
P7 POSITIONS: hilbert_index → absolute PE; trajectory ordinal → sequence PE.
   Both deterministic, both already stored, zero new computation at synthesis time.
   Kills C2. (LapPE literature = external validation that spectral/structural
   PE carries this load.)
P8 SCOPED SYNTHESIS: source/context WHERE-clause conditioning → re-fold → synthesis;
   per-trust-class plane multipliers. Delivers the custom-tailoring product
   AND the persona/register half of conversation (C4).
P9 GEOMETRIC HEADS: angular Gaussian kernel head (exp(−ang²/2σ²), GiST-servable),
   fréchet as first-class shape head, intersects containment mask as structured
   sparsity. Completes 12's Family B.
P10 MEASUREMENT: extend the behavioral harness (verify-model.cmd) with the ONE
   metric that adjudicates everything: routing fidelity at depth — same prompts,
   depth-k grounded-retrieval accuracy + generation fluency vs a conventional-LLM
   floor. Every P-item above lands with its number or it didn't land.
   Suggested order of execution: P1+P4a (conditioning — cheap, global),
   P5 (weights — pure plane math), P3 (allocation), P7 (positions), P6 (vocab +
   discourse plane — the conversational unlock), P2 (nonlinearity), P8, P9.
   P2 is deliberately late: with P1/P3/P4/P5 in place, the linear model's depth
   behavior becomes measurable and the right gate design will be visible.
