## 1. The invention — corrected model of record (from this session's discussion)

### 1.1 Model ingest is ETL, not AI

A trained transformer's weight files are precomputed facts at rest — an
associative database fused with a hardwired query executor (forward-pass
dataflow). ETL extracts the database and discards the executor (§1.5). FFN =
key→value memory, QK = pairwise scoring table, lm_head = hidden→vocab lookup;
the GEMM cascade is a lookup engine re-deriving static facts per call.

### 1.2 Surrogate keys: tokenizer slots and hidden dims

The vocab index is the source's surrogate key; the hidden dim is its internal
basis. Both are joined away at transform. [measured] TinyLlama `tokenizer.json`:
`▁King`=4088, `▁king`=6989, `king`=9292, `▁`=29871, bare `King` ABSENT; vocab
32,000. [measured] `TOKEN_MAPS_TO` = 26,646 rows in `laplace-dev`: 32,000 slots
fold to 26,646 content entities, ratio 0.8327 — and EMBEDS evidence shows
54,571,008 rows / 65,536,000 games = 0.8327 exactly. The fold is real and
self-consistent across two independent measurements.

### 1.3 Content is identity — exact, tiered; transport encodings are source-local spelling (REVISED to as-built)

- Identity = exact content. `king` [k,i,n,g] ≠ `King` [K,i,n,g]: different
  codepoints, different hashes, different entities. No "near-same" exists.
- BPE/SentencePiece markers are declared transport encoding ([measured]
  tokenizer.json normalizer: `Replace " "→"▁"`; decoder: `Replace "▁"→" "`).
- **As built** [verified, `LlamaTokenizerParser.cs:273-303` read this session]:
  `Canonicalize()` strips the marker (`▁` U+2581, `Ġ` U+0120) and **records**
  `TokenRole.LeadingSpace` (never silently discards); GPT-2 `Ċ` decodes to a
  real newline; byte-level `<0xNN>` → raw byte + `ByteLevel` role; bare `▁` →
  single space. So `▁king` and `king` resolve to the **same content entity**
  `king`, with the word-initial framing carried as token-role metadata on the
  vocab record (needed for faithful re-tokenization at synthesis), and the
  per-slot tokenizer entities remain distinct as the source-local namespace.
  Cross-model co-assertion holds: TinyLlama `▁king` and a GPT-2-family `Ġking`
  land on one entity. (My earlier derivation — that ingest should produce a
  5-codepoint `" king"` content entity — described a different resolution than
  what is built; see §2.11.)
- Laplace's tiers are UAX#29-governed and source-independent (codepoint →
  grapheme → word → sentence → document); tokenizer pieces are arbitrary source
  spans connected through the composition DAG. `King` has no vocab slot yet
  exists via decomposition of `▁King`. Case relates through seeded Unicode
  mappings ([measured] `HAS_UPPERCASE_MAPPING` 1,505 rows in laplace-dev) —
  relation, never identity.

### 1.4 Identity hash and S³ geometry (mechanics)

- BLAKE3-128 over canonical strings [verified in demo SQL + KindRegistry]:
  codepoint = `BLAKE3(utf8(char))`; kind = `BLAKE3('substrate/kind/<NAME>/v1')`;
  decomposer source = `BLAKE3('substrate/source/<X>Decomposer/v1')`; **model
  source id = content hash over the model's own files**
  (`SourceEntityIdConventions.cs:145` [verified]). CLI hex = two BE u64s; DB
  stores per-u64 LE.
- S³: atoms by super-Fibonacci (Alexa CVPR 2022 — web-verified;
  `super_fibonacci.c` [agent-audited]); composites = centroid toward origin,
  radius tracks complexity. [measured] `laplace`: tier-0 sample radius =
  1.0000 on all 200k rows; composite radii 1.0→0.0031; max 1,401 constituents.
- Trajectories are `GeometryZM` whose every vertex is a fixed-exponent 4D
  point carrying pure payload [verified, `mantissa.c:11-30,58-83`]: exponent
  pinned to 0x3FF (values in [1,2)), 4×53 payload bits per vertex — **X =
  entity-id lo bits 0-52, Y = id lo 53-63 + id hi 0-41, Z = id hi 42-63 +
  31 flag bits, M = ordinal(16) | run_length(16) | flags(21)**. Identity
  lives in the spatial axes; order lives entirely in the measure axis — so
  same-constituent vertices coincide spatially and differ only in M, and a
  parent's trajectory is literally the sequence of its children's ids. [measured] Coverage is
  **exactly complete in both DBs**: physicalities − trajectories = **1,114,112
  in both** (laplace 9,522,068−8,407,956; laplace-dev 1,140,630−26,518) =
  precisely the atom count — every composite has its trajectory, atoms need
  none.
- [measured] Round-trip: entity `72e7eea2…` → `"Paris"` in 10.6 ms, no model in
  path — **but via session-built `pg_temp` scaffolding, not product surface**
  (§7.4).

### 1.5 Runtime properties = the source's query engine, not knowledge

Softmax→ORDER BY, attention weighting→weighted join, layer stack→recursion,
sampling→LIMIT. The runtime is replaceable and regenerable into any mold shape
from consensus (recipes; `synthesize`), which proves it carries nothing the
substrate lacks. Speed claim status: ms-scale indexed reads [measured 6.7 ms];
full composition not yet practical (§6, §7.3).

### 1.6 Kinds are arenas; one normalization rule at every level

[verified, `KindRegistry.cs:6-44`] (1) different relations → different arenas
(preserve); (2) one relation, two names → one arena (`HAS_UPOS,
HAS_LEX_CATEGORY → HAS_POS` — [measured] single `HAS_POS` arena, 3,335,857
rows, fed by WordNet+OMW+UD); (3) two directions → one arena, flipped
(`IS_HYPERNYM_OF → IS_A`); (4) fine vs coarse → distinct arenas + is_a-on-kinds
roll-up (`DEP_NSUBJ is_a DEPENDS_ON`, `DEP_NSUBJ_PASS is_a DEP_NSUBJ`
[verified `KindRegistry.cs:228-237`]; `ATTENDS/OV_RELATES is_a RELATED_TO`) —
a model is case (4), not a separate category. Dynamic families: the deprel IS
the arena (`DEP_<REL>` uppercased; [measured] 24+ DEP_ and 12+ FEAT_ arenas
live). Model tensor roles are **declared** by architecture → normalize at write
time, modality-blind. Rule in one line: **normalize what the source's declared
structure makes certain at write time; rate what only evidence can tell you.**

### 1.7 Evidence vs consensus; signed Glicko-2

Evidence = provenance (content-addressed PK; outcome 0=refute/1=draw/2=confirm
[measured: model emits 50.0/50.0 split of 0/2 from weight signs];
observation_count = folded games; context mostly NULL). Consensus = one row per
relation, fixed-point 1e9, μ baseline 1500 ([measured] laplace-dev μ ∈
[1192.01, 1812.71], mean 1500.05). Kernel [agent-audited, quotes consistent
with all measurements]: score derived `½(1+tanh(m/M))`
(`AttestationFactory.cs:49-50`), M = measured arena scale (RMS over cells —
[verified] the C# loop at `ModelTableETL.cs:183`, a layer violation §7.5 but
the right math), witness weight → opponent φ ∈ [30,350]
(`AttestationFactory.cs:39-42`) with native `g(φ)` weighting (`glicko2.c`),
deterministic int64/int128 fixed point, no tier-band seeds, no knobs.

### 1.8 Interpretability is a read, not a write

Slice evidence by witness frame, join relata against labeled arenas by content
identity, GROUP BY labels — meaning is derivable on demand, graded, never
asserted; no feedback loops. [measured] `ATTENDS`/`OV_RELATES` = zero ingested
rows: circuits are query-time reads, never ingest materializations. Open
tension flagged in §9.7: with positions folded (by design), per-layer/head
slicing needs the recipe axis — whose persistence I could not locate.

### 1.9 Prompts are ingested as NON-ATTESTED content (NEW this session)

Inference does not "process a prompt token by token." The prompt is **ingested
as content** through the same decomposer path — content-addressed, tiered,
placed, deduped (its known constituents cost nothing) — and immediately *is*
substrate. But it lands in the **content layer only**: no attestations, no
games, no μ movement. **Attested content vs non-attested content** is the
distinction: seeds and models testify; prompts are usage examples. (The code
names this path: `ContentRoundtrip` is the non-attested content path
[agent-audited].) Consequences:
- **No context window exists.** Context limits are artifacts of the executor
  frame (fixed-width buffers). Here "context" = the substrate; everything ever
  ingested is in scope of every query. Conversation history is content in the
  DB, not tokens in a buffer.
- **Inference can originate from any entity** — word, sentence, pixel block,
  relation, verdict — and the response is whatever axes you query (ranked-μ per
  arena, trajectory shape, containment, generation tree), cross-modal.
  `generate_tree(p_prompt bytea, …)` takes an entity id, not a token sequence.
- If a user's corpus ever *should* testify, per-user/tenant source trust is the
  lever — already in the weight law (kind-rank × source-trust × tenant-trust).

### 1.10 Witness classes are individually sufficient (NEW this session)

Datasets-only, models-only, or both reconstruct the same kind of substrate:
corpus→annotation→attestations and corpus→SGD→weights→ETL→attestations are two
routes to the same content×content relations (a model can't testify to what its
corpora didn't contain). "Model vs dataset" is witness frame — out of identity,
into provenance. Inference reads consensus over whatever testimony exists:
- [measured] `laplace` today = seeds-only and fully inferenceable; `laplace-dev`
  = models-only over a reference skeleton and must stand alone — its weak demo-C
  output is *not* a missing-seeds problem (§6).
- [measured] Identity space is shared across classes: model ingest first-observed
  only **34,505 novel entities (~3%)** — 97% of TinyLlama's references already
  existed from the Unicode/ISO skeleton.
- Scoping is a parameter: `attestation_response(..., p_source_scope bytea[])`.
- Validation independence: never use seed agreement as the oracle for model
  ingest, never the source model as the oracle (codec frame, banned); a
  models-only substrate validates by its own coherence under traversal.
- No cold start: one witness suffices; thinness is expressed as RD/volatility.
