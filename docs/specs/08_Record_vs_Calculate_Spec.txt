PRINCIPLE
---------
Ingestion is RECORDING (witnessing), not PROCESSING. Two layers:
  witnessed  = what a source literally asserts. Transcribe verbatim into the 5-tuple.
  calculated = anything derived by interpreting the witnessed layer. Deferred, versioned.
Same content = same hash => linking/meshing is automatic (insert-if-absent), never an
entity-resolution pass. Calculated layer is a pure deterministic fn of witnessed layer =>
memoized, content-addressed, re-derivable, EVICTABLE.

WHY (measured)
--------------
Chess PGN ingest is slow vs WordNet because current ingest REPLAYS a chess engine per ply
(San.Resolve legal-move-gen) and folds consensus inline. WordNet just transcribes. Move the
replay + derivation out of ingest and chess ingest becomes transcription too. Also de-risks
the PCoreParallelCompose drop bug (append-only recording is idempotent/re-runnable) and
[Superseded 2026-07-20: PCoreParallelCompose.cs was deleted (0 hits across app/engine/
extension; removal recorded in .cursor/plans/full_stack_remediation_bdaba5c3.plan.md:115 as
having zero production callers). Compose now runs through the shared pipeline spine
(ComposeDecomposer → IngestPipelineDefaults.Compose, Decomposer.cs:339-348). The named drop
bug is moot with the code.]
subsumes the per-ply waste (EmitNodes 6x, 2M memo cap, AppendEval substructure re-iterate).

RECORDER (new cheap ingest path)
--------------------------------
Records, per game, WITHOUT any chess logic (no board replay, no move gen, no geometry):
  - Game entity (Document) = GameId(white,black,date,moves). Same game => recorded once;
    re-seeing just adds witnesses (played-by/at-time).
  - Tags as witnessed content: Event, Date, ECO, Opening, Result, Termination, TimeControl,
    Site, Round, titles, Annotator (capture MORE tags than today; they're free to record).
  - Players + Elo (HAS_WHITE/BLACK/RATING).
  - Ordered SAN move tokens attributed to the game (the move list as-is; NO resolution).
  - Per-ply witnessed annotations: %clk, %eval token, NAG/glyph, AND free-text comment prose
    (** currently DROPPED — gap to close: emit comment text as a witnessed content node on the
    ply **).
    [Superseded 2026-07-20: GAP CLOSED. ChessPgnDecomposer.cs:197-205 (RecordMovetext) witnesses
    the VERBATIM PGN movetext — clocks, evals, comments, NAGs, result token — as one content
    edge per game (HAS_MOVETEXT); every per-ply token is reconstructible from it via
    MovetextTokens.Parse. Deliberately at game grain, not per ply (see :153-157: per-ply record

  All append-only, content-addressed, idempotent. Source = ChessPgn (witnessed trust class).

ANALYZER (new deferred pass, UI-triggered / batched)
----------------------------------------------------
Pure fn of the witnessed layer. Scans games where analyzed_version < current, then:
  - Replay SAN -> concrete moves -> board positions (the deferred San.Resolve cost).
  - Compose positions (canonical key, substructures, coord/Hilbert/trajectory physicalities).
  - Cross-game meshing happens HERE, deterministically (same position => same hash) and caches
    forever.
  - Motifs, opening classification, engine eval (if run), move-quality inference.
  - Consensus fold (Glicko) over the derived + witnessed outcomes.
  Emits derived attestations under an ANALYSIS source/trust class, stamped analysis_version.
  Idempotent: deterministic => identical ids => re-run dedups. Bump analysis_version to evict
  + re-derive without touching the witnessed layer.

WATERMARK
---------
Per-game analysis_version (stored as a witnessed/meta attestation or a side table). Scan =
"games with analyzed_version < CURRENT_ANALYSIS_VERSION". Growth of novel positions is
sublinear (Heaps ~N^β, not strictly log); consensus update per witness is O(1). First corpus
expensive, rest cheap.

EVAL / QUALITY ARE COMPETING WITNESSES, NOT DUPLICATES (decided 2026-07-04)
---------------------------------------------------------------------------
The PGN's %eval and its ?/! NAGs are a THIRD-PARTY LOWER-TRUST WITNESS to a position's value /
a move's quality — no different from a GM book's assessment. They must COMPETE against what we
calculate, so do NOT dedupe them away. Design:
  - Recorder keeps the PGN eval/quality on the ply node (witnessed, immutable record).
  - Analyzer, at the replayed position, emits TWO attestations on the SAME (position, HAS_EVAL)
    and (position, MOVE_QUALITY): (a) its OWN value under ChessAnalysis (high trust), and (b) the
    PGN's value bridged from the ply node onto the position under the lower-trust annotation
    source (NOT folded into ChessAnalysis).
  - Consensus reconciles by trust: calc dominates, PGN survives as corroboration/provenance, and
    their DIVERGENCE is itself signal (human blunder-call the engine disputes, or vice versa).
  Clock has no calculated rival => provenance-only, stays on the ply. (Corrects an earlier
  "drop the position-attached copies" recommendation, which would have silenced a witness.)

TRUST (see project_calc_trust_model)
------------------------------------
  deterministic-exact calc (position/legality/material/mate/canonical opening) = ground truth,
    top trust, overrides testimony. Immutable until logic changes.
  heuristic calc (eval/motif/opening/quality) = high but f(method: engine, depth,
    analysis_version); supersedable by a better calc. Bind trust to method, don't flat-rate.
  Trust lives in source/trust-class. Different RELATION TYPE only when it's a different KIND of
    claim or must not fold with the witnessed one; SAME relation + different source when
    consensus should reconcile competing estimates (deep eval dominates, annotator eval survives
    as provenance). Calc-outranks-observation only where they claim the SAME fact; provenance
    (who/when/clock) has no calculated rival.

MIGRATION
---------
Existing 126.3M ChessPgn attestations on hart-desktop are today's monolithic witnessed+calculated
mix under one source. The split separates them going forward. Existing rows can be left as-is or
evicted+re-derived once the analyzer exists (per-source eviction, then re-run). Don't re-ingest
without eviction (double-counts consensus — IngestRunner completion-marker guard).
